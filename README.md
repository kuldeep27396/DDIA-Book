> Chapter-by-chapter **study notes and supplementary material** for *Designing Data-Intensive Applications* by **Martin Kleppmann** — the definitive book on reliable, scalable, and maintainable systems.

## Overview

DDIA is widely considered essential reading for data engineers and backend architects. This repository is my structured study companion: for each chapter I collect a study guide, the chapter reference, and related deep-dive papers/articles that reinforce the concepts.

## Why DDIA?

Understanding replication, partitioning, consistency, and batch/stream processing is what separates junior from senior data engineers. These notes distill the book into reviewable, interview-ready material.
